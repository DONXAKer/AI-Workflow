# Название задачи

## Как сейчас
Опишите текущее поведение или проблему. Факты, не оценки.
Что пользователь видит сейчас и почему это плохо.

## Как надо
Целевое поведение и технический подход к реализации.
Можно включать конкретные файлы, компоненты, API-контракты, SQL.

## Вне scope
- Что явно не делаем в этой задаче
- Связанные улучшения, которые оставляем на потом
- Граничные случаи, которые игнорируем

## Критерии приёмки
- [ ] Верифицируемое условие — можно проверить автоматически или вручную
- [ ] Ещё одно условие
